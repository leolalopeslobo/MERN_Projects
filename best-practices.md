VCoding Mistakes:
1. Endpoints have no rate limiting because AI builds your API and leaves it completely open. So someone can hit it thousands of times a second, and you won’t know until your server crashes or your bill becomes insane. So make sure to add rate-limiting middleware, and if you're using Express, it is super easy to set up.
2. RLS - Row Level Security is off by default in Supabase. Auth will work fine, users are logging in, and everything looks good, but anyone can query your database and pull other people’s data because RLS is not enabled. So always turn it on for every table and write policies that scope data to the right user.
3. This is an “insecure direct object reference” because AI will generate routes like this: /api/user/123, and you will never question it. But any logged-in user can just change that 123 to someone else’s ID and access their data. Therefore, always verify that the person making the request actually owns what they are asking for.
4. There's also versioning of APIs that should be done - explain if this is really important or not and why. (Backward compatibility)
